🚀 API ENDPOINTS STRUCTURE
───────────────────────────────────────────────────────────────────────────

POST /api/admin/login
├─ Public endpoint (but checks role)
├─ Body: { email, password }
├─ Returns: { token, user with role }
└─ Response: 200 or 403 (if not admin)

GET /api/admin/users
├─ Requires: Auth + superadmin role
├─ Query params: page, limit, role
├─ Returns: Paginated user list
└─ Response: 200 or 403

PUT /api/admin/users/:userId/role
├─ Requires: Auth + superadmin role
├─ Body: { role }
├─ Returns: Updated user
└─ Response: 200 or 403

GET /api/admin/stats
├─ Requires: Auth + admin/superadmin role
├─ Returns: Dashboard statistics
└─ Response: 200 or 403

⚙️ MIDDLEWARE CHAIN
───────────────────────────────────────────────────────────────────────────

Request Flow:
1. Client sends request with Authorization header
2. auth middleware:
   ├─ Extract token from header
   ├─ Verify JWT signature
   ├─ Find user by userId in token
   └─ Attach user to req.user

3. authorize middleware:
   ├─ Check if req.user.role is in allowed roles
   ├─ If yes: proceed to handler
   └─ If no: return 403 Forbidden

4. Route handler executes with authorized user

⚡ QUICK START SEQUENCE
───────────────────────────────────────────────────────────────────────────

Step 1: Run Seed Script
$ node scripts/seedSuperadmin.js
Result: Superadmin created with credentials

Step 2: Login
$ curl -X POST /api/admin/login -d '{"email":"admin@...", "password":"..."}'
Result: JWT token returned

Step 3: Use Token
$ curl -X GET /api/admin/users -H "Authorization: Bearer <token>"
Result: User list returned (if superadmin)

Step 4: Protect Routes
router.post('/api/products',
  auth,

);
